# Localization

Under the hood we are using [i18next](https://www.i18next.com/) to handle localization
of various parts of our search interfaces. We export our instance of [i18next](https://www.i18next.com/)
so that it is possible to provide your own localizations for your search interfaces.

By default we load the relevant localization dictionary based on the a user's browser settings.

## Usage
```javascript
import { i18n } from "@sajari/sdk-react"

// https://www.i18next.com/how-to/add-or-load-translations#add-after-init

// Add french localizations for the summary component
i18n.addResource("fr", "summary", {
    page: "Page {{pageNumber}} de",
    resultsFor: "r√©sultats pour",
    searchInstead: "Essayez avec l'orthographe",
})

```

## Namespaces
Internally we have used namespaces to denote different components,
currently we only support 2 namespaces:

1. `errors`
1. `summary`

### Errors
```json
errors: {
    authorization: "Authorization for this request failed. Check your credentials.",
    connection: "Please check your network connection.",
    parseResponse: "Unable to parse response."
}
```

### Summary
```json
summary: {
    page: "Page {{pageNumber}} of",
    resultsFor: "results for",
    searchInstead: "search instead for"
}
```
